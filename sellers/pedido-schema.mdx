Esta página descreve a estrutura do objeto **Order** (`$id: https://schema.mkplace.com.br/sales/order.json`) que a Mkplace envia ao seller a cada evento de pedido. Abaixo estão os campos da raiz e os blocos mais importantes; o JSON Schema completo está disponível para download.

## Campos da raiz

| Campo | Tipo | Obrigatório | Descrição |
| --- | --- | --- | --- |
| `_id` | string | Sim | Identificador do pedido na Mkplace. |
| `summary` | object | Sim | Resumo do pedido: quantidade de itens, totais, descontos e prazos. |
| `deliveryAddress` | object | Sim | Endereço de entrega do pedido. |
| `customer` | object | Sim | Dados do cliente final (nome, contato, documento). |
| `items` | array | Sim | Itens do pedido (SKU, quantidade, preços, frete por item). |
| `parentOrderId` | string | Não | Id do pedido que originou o atual (quando aplicável). |
| `discounts` | array | Não | Descontos aplicados ao pedido (ex.: cupom). |
| `shipments` | array | Não | Pacotes do pedido — surgem a partir do faturamento (nota fiscal, tracking, status e itens). |
| `transactionId` | string | Não | Id da transação de pagamento. |
| `verifyToken` | string | Não | Representação do pedido em forma encriptada. |
| `exceptionDetails` | string | Não | Mensagem com informação de erro, quando houver. |
| `promotions` | array \| null | Não | Informações de promoções. |
| `event` | object \| null | Não | Informação do evento que originou a notificação (`code`, `description`). |
| `metadata` | object \| null | Não | Dados adicionais livres. |

<Note>
  O conceito de **status de pedido, pacote e item** (incluindo o modelo multi-seller) é explicado em [Pedidos](/sellers/pedidos). Os status possíveis aparecem nos blocos `shipments[].status` e `shipments[].items[].status` abaixo.
</Note>

## Blocos principais

<AccordionGroup>
  <Accordion title="summary — resumo do pedido" icon="receipt">
    ```json
    "summary": {
      "type": "object",
      "properties": {
        "items": { "type": "number", "minimum": 1, "description": "SKU quantity" },
        "total": { "type": "number", "minimum": 0, "description": "Sum of all prices." },
        "totalPriceDiscount": { "type": "number", "minimum": 0, "default": 0, "description": "Sum of all items' discounts." },
        "totalShippingCost": { "type": ["number", "null"], "minimum": 0, "default": 0, "description": "Sum of all freight costs." },
        "totalShippingDiscount": { "type": ["number", "null"], "minimum": 0, "default": 0, "description": "Sum of all freight related discounts." },
        "maxShippingTime": { "type": ["number", "null"], "minimum": 0, "default": 0, "description": "Order's max shipping time" },
        "ordersDiscount": { "type": ["number", "null"], "minimum": 0, "default": 0, "description": "A discount that applies to the whole order." },
        "finalPrice": { "type": "number", "minimum": 0, "default": 0, "description": "The final price of the whole order." }
      },
      "required": ["items", "total", "totalPriceDiscount"]
    }
    ```
  </Accordion>

  <Accordion title="items[] — itens do pedido (campos obrigatórios)" icon="box">
    Cada item do array `items` traz, no mínimo, os campos abaixo. Veja o restante (frete por item, descontos unitários, `warehouses`, `attributes`) no schema completo.

    ```json
    "items": {
      "type": "array",
      "minItems": 1,
      "items": {
        "type": "object",
        "required": [
          "offerId", "skuId", "productName", "productSlug", "slug", "name",
          "quantity", "sellerName", "sellerId", "sellerSkuId",
          "price", "priceDiscount", "originalPrice", "unitPrice", "unitOriginalPrice", "attributes"
        ]
      }
    }
    ```
  </Accordion>

  <Accordion title="shipments[] — pacotes, nota fiscal e tracking" icon="truck">
    Os pacotes surgem com o faturamento. Cada `shipment` tem `status`, `billing` (nota fiscal) e `items` obrigatórios, além de `tracking` e `event` opcionais.

    ```json
    "shipments": {
      "type": "array",
      "items": {
        "type": "object",
        "properties": {
          "status": {
            "type": "string",
            "enum": ["BILLED", "SENT", "DELIVERED", "CANCELED", "REPLACED", "REFUNDED", "RETURNED"]
          },
          "items": {
            "type": "array",
            "items": {
              "properties": {
                "status": {
                  "type": "string",
                  "enum": [
                    "BILLED", "SENT", "DELIVERED",
                    "PARTIALLY-BILLED", "PARTIALLY-SENT", "PARTIALLY-DELIVERED", "PARTIALLY-RETURNED",
                    "CANCELED", "REPLACED", "REFUNDED", "RETURNED"
                  ]
                }
              }
            }
          }
        },
        "required": ["status", "billing", "items"]
      }
    }
    ```
  </Accordion>
</AccordionGroup>

A partir dessas informações, o vendedor consegue armazenar o pedido para faturá-lo, enviá-lo e executar as demais ações descritas em [Pedidos](/sellers/pedidos).
